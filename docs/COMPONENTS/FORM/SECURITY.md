## XSS Protection

### Automatic Escaping

All user-controllable data is automatically escaped using `htmlspecialchars()` with proper flags:

```php
function canvastack_form_escape_html($string): string {
    if (is_null($string) || false === $string) {
        return '';
    }
    return htmlspecialchars((string)$string, ENT_QUOTES | ENT_HTML5, 'UTF-8');
}
```

**What is Escaped:**
- Form field names
- Field values
- Attribute values
- Label text
- Error messages
- Option labels and values
- Tab labels
- File paths for display

### SafeHtml Marker System

To prevent double-encoding while maintaining security, the Form Builder uses a SafeHtml marker system:

```php
// Mark HTML as safe (already escaped)
$html = SafeHtml::mark('<div class="form-control">Safe HTML</div>');

// Check if HTML is marked
if (SafeHtml::isMarked($html)) {
    // Don't escape again
}

// Process HTML (unmark if safe, escape if not)
$output = SafeHtml::process($html);
```

**When to Use:**
- ✅ Use for HTML generated by trusted internal functions
- ✅ Use after escaping user input
- ❌ Never use for raw user input
- ❌ Never use for external data

### Dangerous Attribute Blocking

Event handlers and dangerous attributes are automatically blocked:

```php
// Blocked attributes
$dangerous = [
    'onclick', 'onload', 'onerror', 'onmouseover',
    'onfocus', 'onblur', 'onchange', 'onsubmit',
    'javascript:', 'data:text/html'
];

// Example
$form->text('username', null, [
    'onclick' => 'alert(1)' // ❌ Blocked!
], true);
```

**Security Warning:** ⚠️ Never disable attribute validation!

## File Upload Security

### Multi-Layer Validation

File uploads go through multiple validation layers:

#### 1. Extension Whitelist

```php
protected array $allowedFileExtensions = [
    'jpg', 'jpeg', 'png', 'gif', 'webp', 'svg',  // Images
    'pdf', 'doc', 'docx', 'xls', 'xlsx',         // Documents
    'txt', 'csv', 'zip', 'rar'                   // Other
];
```

**Security:** Prevents execution of malicious files (php, exe, sh, etc.)

#### 2. MIME Type Validation

```php
// Validates actual file content, not just extension
$mimeType = $file->getMimeType();
if (!in_array($mimeType, $allowedMimeTypes)) {
    throw new \InvalidArgumentException('File type not allowed');
}
```

**Security:** Prevents extension spoofing (e.g., malicious.php.jpg)

#### 3. Double Extension Check

```php
// Detects: file.php.jpg
$filenameParts = explode('.', $filename);
if (count($filenameParts) > 2) {
    // Check for dangerous extensions
    $dangerousExtensions = ['php', 'phtml', 'exe', 'sh', 'bat', 'js'];
    // Block if found
}
```

**Security:** Prevents double extension attacks

#### 4. File Size Validation

```php
protected int $maxFileSize = 10485760; // 10MB

if ($fileSize > $maxFileSize) {
    throw new \InvalidArgumentException('File too large');
}
```

**Security:** Prevents DoS attacks via large files

#### 5. Random Filename Generation

```php
private function generateSecureFilename(string $originalFilename): string {
    $extension = strtolower(pathinfo($originalFilename, PATHINFO_EXTENSION));
    $randomName = bin2hex(random_bytes(16));
    $timestamp = time();
    
    return "{$timestamp}_{$randomName}.{$extension}";
}
```

**Security:** Prevents overwrite attacks and filename-based exploits

#### 6. File Permissions

```php
chmod($filePath, 0644); // Read-only for group/others
```

**Security:** Prevents execution of uploaded files

### File Upload Example

```php
// Configure allowed types
$form->allowedFileExtensions = ['jpg', 'jpeg', 'png'];
$form->allowedMimeTypes = ['image/jpeg', 'image/png'];
$form->maxFileSize = 2097152; // 2MB

// Process upload
try {
    $form->fileUpload('uploads/avatars', $request, [
        'input_name' => 'avatar',
        'validation' => 'required|image|max:2048',
        'generate_thumbnail' => true
    ]);
} catch (\InvalidArgumentException $e) {
    // Handle validation error
    return back()->withErrors(['avatar' => $e->getMessage()]);
}
```

**Security Warnings:**
- ⚠️ Never trust file extensions alone
- ⚠️ Always validate MIME type from content
- ⚠️ Never store uploads in web-accessible directories without validation
- ⚠️ Always use random filenames
- ⚠️ Set proper file permissions (0644)

## Path Traversal Prevention

### Upload Path Validation

```php
private function validateUploadPath(string $path, string $baseDir): bool {
    // Normalize paths
    $realPath = realpath($path) ?: $path;
    $realBase = realpath($baseDir) ?: $baseDir;
    
    // Check for traversal patterns
    if (str_contains($path, '..') || str_contains($path, './')) {
        throw new \InvalidArgumentException('Directory traversal detected');
    }
    
    // Ensure path is within base directory
    if (!str_starts_with($realPath, $realBase)) {
        throw new \InvalidArgumentException('Path outside allowed directory');
    }
    
    return true;
}
```

**Blocked Patterns:**
- `../` - Parent directory
- `./` - Current directory
- `..\` - Windows parent directory
- `.\` - Windows current directory

### Example Attack Prevention

```php
// ❌ Attack attempt
$form->fileUpload('../../etc/passwd', $request, $fileInfo);
// Result: InvalidArgumentException thrown

// ✅ Safe path
$form->fileUpload('uploads/avatars', $request, $fileInfo);
// Result: Upload proceeds
```

**Security Warnings:**
- ⚠️ Never construct paths from user input without validation
- ⚠️ Always use `realpath()` to resolve symbolic links
- ⚠️ Validate paths against trusted base directories

## Input Validation

### Attribute Validation

```php
function canvastack_form_validate_attributes(array $attributes): array {
    $dangerous = [
        'onclick', 'onload', 'onerror', 'onmouseover',
        'onfocus', 'onblur', 'onchange', 'onsubmit',
        'onkeydown', 'onkeyup', 'onkeypress'
    ];
    
    foreach ($attributes as $key => $value) {
        if (in_array(strtolower($key), $dangerous)) {
            throw new \InvalidArgumentException(
                "Dangerous attribute not allowed: {$key}"
            );
        }
    }
    
    return $attributes;
}
```

### Validation Rule Propagation

```php
// Server-side validation
$form->setValidations([
    'email' => 'required|email|max:255',
    'age' => 'required|numeric|min:18|max:120'
]);

// Automatically adds client-side attributes
// email: required, type="email", maxlength="255"
// age: required, type="number", min="18", max="120"
```

**Security:** Provides defense in depth with both client and server validation

## Encryption Security

### Model Name Encryption

```php
// Encrypt model name with integrity check
private function generateEncryptedModelName(?object $model): string {
    $modelName = get_class($model);
    $hash = hash_hmac('sha256', $modelName, config('app.key'));
    $encrypted = encrypt($modelName . '|' . $hash);
    
    return $encrypted;
}

// Validate before decryption
private function validateEncryptedModelName(string $encrypted): string|false {
    try {
        $decrypted = decrypt($encrypted);
        [$modelName, $hash] = explode('|', $decrypted);
        
        $expectedHash = hash_hmac('sha256', $modelName, config('app.key'));
        
        if (!hash_equals($expectedHash, $hash)) {
            throw new \Exception('Integrity check failed');
        }
        
        return $modelName;
    } catch (\Exception $e) {
        error_log('SECURITY WARNING: Model name tampering detected');
        return false;
    }
}
```

**Security:** Prevents model name tampering and injection attacks

### AJAX Query Encryption

```php
// Encrypt query for sync() method
$query = 'SELECT id, name FROM cities WHERE country_id = ?';
$encrypted = encrypt($query);

$form->sync('country_id', 'city_id', 'id', 'name', $encrypted, null);

// Server-side validation
private function validateSyncQuery(string $encrypted): string {
    $query = decrypt($encrypted);
    
    // Check for dangerous SQL patterns
    $dangerous = ['DROP', 'DELETE', 'TRUNCATE', 'ALTER', 'CREATE'];
    foreach ($dangerous as $pattern) {
        if (stripos($query, $pattern) !== false) {
            throw new \SecurityException('Dangerous SQL detected');
        }
    }
    
    return $query;
}
```

**Security:** Prevents SQL injection in AJAX relational fields

## CSRF Protection

### Automatic Token Inclusion

```php
// CSRF token automatically added by open()
$form->open('/users/store', 'POST');
// Includes: {{ csrf_field() }}

// Manual token insertion
$form->token();
```

### Token Validation

Laravel automatically validates CSRF tokens for POST, PUT, PATCH, DELETE requests.

**Security Warnings:**
- ⚠️ Never disable CSRF protection
- ⚠️ Always use POST for state-changing operations
- ⚠️ Never expose CSRF tokens in URLs

## Security Best Practices

### 1. Always Validate User Input

```php
// ✅ Good
$form->setValidations([
    'email' => 'required|email|max:255',
    'age' => 'required|numeric|min:18'
]);

// ❌ Bad - no validation
$form->email('email');
```

### 2. Use Type Hints

```php
// ✅ Good - type safety
public function store(Request $request): RedirectResponse {
    $validated = $request->validate([...]);
    User::create($validated);
}

// ❌ Bad - no type safety
public function store($request) {
    User::create($request->all()); // Mass assignment vulnerability!
}
```

### 3. Sanitize File Uploads

```php
// ✅ Good
$form->allowedFileExtensions = ['jpg', 'png'];
$form->allowedMimeTypes = ['image/jpeg', 'image/png'];
$form->maxFileSize = 2097152; // 2MB

// ❌ Bad - accepts any file
$form->fileUpload('uploads', $request, $fileInfo);
```

### 4. Use HTTPS

```php
// ✅ Good - force HTTPS
if (!$request->secure()) {
    return redirect()->secure($request->getRequestUri());
}

// ❌ Bad - allows HTTP
// Passwords and sensitive data transmitted in plain text!
```

### 5. Log Security Events

```php
// Log suspicious activity
error_log('SECURITY WARNING: XSS attempt detected: ' . $input);
error_log('SECURITY WARNING: Path traversal attempt: ' . $path);
error_log('SECURITY WARNING: Invalid file upload: ' . $filename);
```

### 6. Keep Dependencies Updated

```bash
# Regularly update dependencies
composer update

# Check for security vulnerabilities
composer audit
```

### 7. Use Content Security Policy

```php
// Add CSP header
header("Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'");
```

### 8. Implement Rate Limiting

```php
// Protect against brute force
Route::post('/login')->middleware('throttle:5,1');
```

## Security Checklist

Before deploying to production:

- [ ] All user input is validated
- [ ] File uploads are restricted to safe types
- [ ] CSRF protection is enabled
- [ ] HTTPS is enforced
- [ ] Passwords are hashed (bcrypt)
- [ ] Sensitive data is encrypted
- [ ] Error messages don't expose sensitive information
- [ ] Security headers are configured
- [ ] Rate limiting is implemented
- [ ] Dependencies are up to date
- [ ] Security logging is enabled
- [ ] Backup and recovery plan exists
