## 📝 Documentation Standards

### File Naming
- Use kebab-case: `alpine-js.md`, `dark-mode-toggle.md`
- Be descriptive: `creating-components.md` not `components.md`
- Use singular form: `component.md` not `components.md` (except for overview files)

### Document Structure
Each document should include:
1. **Title** - Clear, descriptive title
2. **Location** - File/component location
3. **Features** - Key features list
4. **Basic Usage** - Simple example
5. **Props/API** - Parameters and options
6. **Examples** - Multiple use cases
7. **Implementation** - Technical details
8. **Testing** - Test examples
9. **Tips** - Best practices
10. **Related** - Related components/docs
11. **Resources** - External links

### Code Examples
- Use syntax highlighting
- Include complete, runnable examples
- Show both Blade and JavaScript when applicable
- Provide multiple variants/use cases

### Metadata
Each document should end with:
```markdown
---
**Last Updated**: YYYY-MM-DD  
**Component Version**: X.Y.Z
```
